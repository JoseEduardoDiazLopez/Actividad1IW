Control de Versiones



¿Qué es un control de versiones, y por qué debería importarte?

Es necesario almacenar las diferentes versiones de un sistema en archivos en sus fases de desarrollo, para tener la posibilidad de regresar a versiones anteriores según la necesidad que se tenga ya sea por un diseño anterior o alguna función en el código que solo se encuentra en una versión anterior.

Además, en caso de provocar algún error en la versión actual, siempre se podrá regresar a una versión funcional, todo esto solo por saber versionar un sistema.

Sistemas de Control de Versiones Locales

Este método consiste en simplemente ir guardando de manera local en la misma computadora las diferentes versiones de un sistema, existen herramientas para hacerlo más eficiente de manera que una persona no lo haga manualmente, como el caso de RCS que se sigue utilizando en la actualidad para versionar.

Sistemas de Control de Versiones Centralizados

Este método ofrece varias características, como que los participantes pueden saber en que trabajan los demás en el proyecto, los administradores pueden asignar tareas y permisos diferentes a cada colaborador. Otra ventaja que ofrece es que no está todo en un mismo lugar y existe menor riesgo de perder todas las versiones.

Sistemas de Control de Versiones Distribuidos

Este tipo de sistemas se encarga de manejar repositorios de manera remota, donde simultáneamente los colaboradores pueden trabajar en un mismo proyecto desde lugares diferentes, esto ayuda a crear distintos ritmos de trabajo que no serían posibles en los otros modelos de control de versiones. Otra ventaja de este modelo es que, al estar un repositorio clonado, los colaboradores no dependen de un solo servidor, de manera que, si cae, alguien puede restaurar el proyecto.

Conclusión

Los distintos modelos de control de versiones ofrecen características distintas, algunas mejores que otras dependiendo de las necesidades actuales, según lo que leí, el control de versiones distribuida me parece la más interesante y segura, con múltiples usos y distintas formas de trabajar ya sea de manera remota o presencial.
